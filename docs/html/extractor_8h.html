extractor.h - container for all file extraction functions.

All functions return zero on success, non-zero on failure. Extracts a file using its respective directory_record and data pointed to by fptr, outputting it to the location denoted by path.

the file_identifier terminates with a ; character followed by the file ID number in ASCII coded decimal (1). See: https://wiki.osdev.org/ISO_9660#Directories

Just use the default/existing filename. It'll be incorrect, but probably not worth stoping execution over. Users could just manually remove the ;1 part; the data itself should be fine.
