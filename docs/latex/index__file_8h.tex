\doxylink{index__file_8h}{index\+\_\+file.\+h} -\/ \doxylink{structindex__file}{index\+\_\+file} datatype and its associated functions. 

Used for processing Harvester\textquotesingle{}s {\ttfamily INDEX.\+00\#} files. 

Both the \doxylink{structindex__file}{index\+\_\+file} datatype and the functions contained herein are based on my own observations from studying the {\ttfamily INDEX.\+00\#} files and their associated {\ttfamily .DAT} files in a hexeditor. 

Through these observations alone, I believe we have enough data to successfully extract files, however, it\textquotesingle{}s worth noting that there is a fair bit of data that we\textquotesingle{}re ignoring here. 

Between the {\ttfamily filename} and {\ttfamily start}, there\textquotesingle{}s occasionally some data, but I can\textquotesingle{}t for the life of me seem to figure out what it\textquotesingle{}s used for. It could be random garbage generated by whatever program created these {\ttfamily .DAT}/{\ttfamily INDEX.\+00\#} files, or they could be file flags, or perhaps even some sort of primative checksum; I\textquotesingle{}m not sure. 

If you have any idea what this data might be used for, feel free to make whatever changes are needed! Both of these are probably too large, but better to be safe than sorry.
